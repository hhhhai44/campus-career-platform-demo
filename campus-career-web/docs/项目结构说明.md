# 项目结构说明（campus-career-web）

本项目为 **Vue 3 + Vite + TypeScript** 的前端工程模板，内置 **Vue Router**（路由）与 **Pinia**（状态管理），并包含 **ESLint/Prettier/Oxlint** 等开发期质量工具。

## 目录/文件一览与作用

### 根目录

- **`README.md`**：项目说明文档（建议团队把开发/构建/部署约定都写这里）。
- **`docs/`**：项目文档目录（放结构说明、约定、接口文档链接、部署说明等）。
  - **`docs/项目结构说明.md`**：本文件，集中解释“每个文件/文件夹作用”与“可删除项”。
- **`package.json`**：Node 依赖与脚本入口（`dev/build/type-check/lint/format`）。  
  - 说明：该文件为 JSON，**不能写注释**，所以建议在 `docs/项目结构说明.md` 或 `README.md` 解释关键脚本/依赖用途。
- **`package-lock.json`**：锁定依赖版本，确保可复现安装。  
  - 建议：一般**不要手改**；提交到 Git（团队协作时很重要）。
- **`index.html`**：Vite 应用的 HTML 入口（挂载点 `#app`，加载 `/src/main.ts`）。
- **`vite.config.ts`**：Vite 构建/开发服务器配置（插件、别名、构建策略等）。
- **`env.d.ts`**：为 Vite 注入的类型声明（解决 TS 对 `import.meta.env`、`.vue` 等的类型识别）。
- **`eslint.config.ts`**：ESLint（含 Vue/TS）配置入口（配合 `npm run lint:*`）。
- **`.oxlintrc.json`**：Oxlint 配置文件（`npm run lint:oxlint` 用）。
- **`tsconfig.json`**：TypeScript “工程引用”入口文件（聚合 app/node 两套 tsconfig）。
- **`tsconfig.app.json`**：浏览器端（Vue 应用代码）TS 配置。
- **`tsconfig.node.json`**：Node 端（Vite 配置、ESLint 配置等）TS 配置。
- **`node_modules/`**：依赖安装目录（自动生成）。  
  - 说明：**不应提交到 Git**（一般由 `.gitignore` 排除）。

### `public/`

- **`public/favicon.ico`**：站点图标，打包时会原样拷贝到产物根目录。  
  - 可选：如果你后续会换成自己的图标/或用 SVG/或由后端注入，也可以替换或移除（但需同步修改 `index.html` 里的 `<link rel="icon"...>`）。

### `src/`（核心业务代码）

- **`src/main.ts`**：应用入口：创建 Vue App、注册 Pinia/Router、挂载到 `#app`。
- **`src/App.vue`**：根组件（路由布局/全局 UI 通常从这里开始）。
- **`src/router/index.ts`**：路由实例创建处（`createRouter`、`routes` 配置）。
- **`src/stores/`**：Pinia store 目录（建议按业务域拆分）。
  - **`src/stores/counter.ts`**：模板示例 store（当前未被引用）。

## 哪些“后期可能不会用 / 可删除”（含条件）

以下都属于“模板/可选工具”，**不是必须删**；删除前请按条件确认。

- **`src/stores/counter.ts`（可删除）**  
  - 条件：项目没有引用 `useCounterStore`（当前模板默认也没引用）。  
  - 删除后影响：无（仅少一个示例）。

- **`src/stores/` 目录（可删除）**  
  - 条件：你决定不使用 Pinia（同时要改 `src/main.ts` 移除 `createPinia()` 注册，并从 `package.json` 移除 `pinia` 依赖）。  
  - 删除后影响：失去全局状态管理能力（可替换为 Vue 自带方案或其他库）。

- **`vue-router`（可选）与 `src/router/`（可删除）**  
  - 条件：你确定项目是单页单视图，不需要路由（同时要改 `src/main.ts` 移除 `app.use(router)` 并从 `package.json` 移除 `vue-router`）。  
  - 删除后影响：无法做多页面/多路由导航。

- **`vite-plugin-vue-devtools`（可选，建议仅开发保留）**  
  - 条件：你不想在开发期集成该插件（同时要改 `vite.config.ts` 移除 `vueDevTools()`，并从 `package.json` 移除依赖）。  
  - 删除后影响：仅影响开发调试体验，不影响生产功能。

- **`eslint.config.ts`、`.oxlintrc.json`（可选，但强烈建议保留）**  
  - 条件：你们团队不用 ESLint/Oxlint（同时要从 `package.json` 移除相关脚本和依赖）。  
  - 删除后影响：代码质量约束下降、CI 可能失效（如果后续加了）。

## 推荐的后续组织方式（建议）

- 业务页面/路由：`src/views/`、`src/router/routes.ts`
- 通用组件：`src/components/`
- API 请求封装：`src/api/`
- 工具函数：`src/utils/`
- 类型：`src/types/`
